The following tools are required in order to develop, build , test, run and package OpenCOR:

  • Git: a distributed version control system;
  • CMake 3.2 or later: a cross-platform build system;
  • Ninja (optional): a small build system with a focus on speed;
  • C++ toolchain: a set of C++ tools. The toolchain is different depending on the platform you are using:
    • Windows: Visual Studio Community 2015;
    • Linux: GCC/G++ 4.9+ (g++-4.9 package or manual installation) and SSL support (libssl-dev package) on Ubuntu; and
    • macOS: Xcode 8.1 (including its command line tools) and SSL support (openssl formula from Homebrew).
  • Qt 5.6.2 LTS: a cross-platform application and UI framework (Qt Creator 4.1.0 is its supporting IDE):
    • Windows (64-bit);
    • Linux (64-bit); and
    • macOS (you might want to install Qt under /Applications rather than your home folder).
  • NSIS 3.0 (optional and Windows only): a program for creating Windows installers.
  • PackageMaker (optional and macOS only): a program for creating macOS installers.

Finally, you will need to update your PATH environment variable by adding the following path(s):

  • Windows: C:\Program Files\Git\bin, C:\Program Files\CMake\bin, C:\Qt\5.6\msvc2015_64\bin and C:\Qt\Tools\QtCreator\bin;
  • Linux: /opt/Qt/5.6/gcc_64/bin in /etc/environment on Ubuntu; and
  • macOS: /Applications/Qt/5.6/clang_64/bin in /etc/profile.

On macOS, you will also need to export OPENSSL_ROOT_DIR=/usr/local/opt/openssl.
